# Capítulo III: Requirements Specification

## 3.1. User Stories

Las User Stories de Nexa se organizan según el flujo funcional del producto y mantienen trazabilidad con las épicas que agrupan las capacidades principales de la solución. Esta organización permite relacionar las necesidades de los segmentos objetivo con funcionalidades verificables mediante criterios de aceptación redactados en formato Gherkin resumido.
